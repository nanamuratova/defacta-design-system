Foundations

Color tokens

Defacta's palette is built for trust, not decoration. A 12-step neutral scale derived from a single ink (#1A1A1A) carries almost the entire UI. Semantic colors appear only to signal meaning — error, warning, success, info — each as a base, a soft fill, and a border. One restrained blue accent is used sparingly. The result reads calm, precise, and evidence-first.

Single source of truth. Every color below is a var(--token) defined in tokens.css. Use the token, never the raw hex — change a value there once and it updates everywhere.

Usage rules

Use
Reference every color through a var(--token) from tokens.css.
Carry structure with the neutral scale: text --grey-950 / --grey-600 / --grey-400, borders --grey-200, surfaces --canvas / --paper.
Use semantic color only to signal state or verdict — pair the base (text / icon / dot) with its soft fill and border.
Use the accent (--color-accent-blue) sparingly: focus rings, eyebrow labels, small highlights.
Don't
Hardcode hex values or use var(--token, #fallback) fallbacks.
Use semantic or accent color for decoration — color must mean something.
Introduce new hues or a second accent.
Put text on a saturated semantic fill — use the matching *-soft tint behind it.

Typography

Defacta sets text in two typefaces: Inter for the interface and IBM Plex Mono for labels, data, and identifiers. The full scale: one display size, four heading levels (H1–H4), five body sizes (XLarge → XSmall), and three sizes each of label and mono/data. Headings and body share two weights — default Medium / Regular and strong Semibold — tighten their tracking as they grow, and the uppercase mono label is the system's signature.

Two typefaces, one rhythm. Inter (400 / 500 / 600) carries all UI text. IBM Plex Mono is reserved for the uppercase .label, data, IDs, and timestamps — never for body copy. Every style is a class in components/base.css driven by tokens in tokens.css.

Usage rules

Use
Build hierarchy with the type classes (.display, h1–h3, .body) — never a manual font-size.
Set running copy in .body / .body-lg; drop to --text-secondary for supporting text.
Use .label for eyebrows & field labels, and .mono for IDs, timestamps, and hex.
One display per screen — let size and whitespace, not weight, carry emphasis.
Don't
Set raw font-size or font-family — use a type class or token.
Use IBM Plex Mono for body text or long copy — it's for labels & data only.
Add weights beyond Regular / Medium / Semibold, or italics outside headings.
Use ALL CAPS anywhere except the mono .label.

Shadows, radius & spacing

Three scales give the system its physical rhythm. Elevation is soft and low — five steps from a hairline lift to a floating overlay. Corners are editorial — a 2→24px radius scale (plus a full/circular option for user elements), with 8–16px covering most UI. Spacing is an 8px-base scale from 0 to 80px (--space-0 → --space-1000, named as the base-unit multiplier ×100) that drives every gap, padding, and margin, so layout always lands on a consistent grid.

Usage rules

Use
Set every gap, padding, and margin with --space-* tokens — stay on the 8px grid.
Use --radius-8 for inputs & buttons, --radius-12/16 for cards, --radius-full for pills.
Reach for the lowest elevation that reads: --shadow-sm cards, --shadow-md menus, --shadow-lg/xl overlays.
Don't
Use arbitrary px values for spacing, radius, or shadows — always a token.
Stack heavy shadows or use elevation as decoration.
Mix different radii within a single component.

Usage rules

Use
Place icons inline with <i data-lucide="name"></i>, then call lucide.createIcons().
Size with .icon--sm (16) / default (20) / .icon--lg (24); color follows text — set it on the parent.
Pick by meaning and reuse the same icon for the same action everywhere.
Don't
Mix icon sets — Lucide only (no emoji, no Iconoir, no Font Awesome).
Use filled or duotone icons, or recolor them for decoration.
Resize with manual width/height — use the size classes.
